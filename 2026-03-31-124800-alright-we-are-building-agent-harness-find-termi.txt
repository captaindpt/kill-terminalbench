  Key Findings

  Two ways to integrate an agent:
  1. Installed agent (like Claude Code) — gets npm-installed inside the task
  Docker container, runs as a CLI command. Full filesystem access.
  2. External agent (like Terminus) — sits outside, sends keystrokes to tmux,
  reads terminal output. Zero contamination.

  How Claude Code runs on TB:
  - Installs itself inside the container
  - Runs claude -p <instruction> with allowed tools: Bash, Edit, Write, Read,
  Glob, Grep, etc.
  - Single shot, blocks until done

  Terminus baseline:
  - 50-episode loop: LLM sees terminal state → outputs structured JSON (commands
   + is_task_complete) → sends keystrokes → repeat
  - Very bare-bones, no tools beyond raw terminal

  Task format: Each task = Dockerfile + instruction + test script + reference
  solution. ~89 hard tasks across SWE, ML, security, sysadmin, data science.

  Attack Vectors to Beat Claude Code

  - Better prompting / reasoning strategy per domain
  - Smarter error recovery and retry logic
  - Multi-model routing (best model per task type)
  - Domain-specific tool scaffolding
  - Better timeout/episode management
  - Can go installed-agent or external-agent path

  What direction do you want to take? Build an installed agent (like Claude Code
   but better orchestration) or an external agent (like Terminus but way
  smarter)?
